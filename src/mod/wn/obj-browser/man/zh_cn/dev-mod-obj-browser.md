# 什么是对象浏览模型

简单的浏览目录，通过 `obj-current` 模型就能办到。
但是我们想在浏览对象时提供更多更复杂的配置。 所以就有了这个模型。

首先，当前的`meta`不再是目标源目录，而是一个配置项，内容就是本模型的字段（除了`meta`）

```js
{
  // 模型的 obj 元数据
  "meta": null,
  // 是否本地保存用户最后一次设置的过滤条件
  "keepSearch": true,
  //--------------------------------------------------
  // 下面的这些字段都是来自 meta 对象的 JSON　内容
  //--------------------------------------------------
  // 指向目标目录
  "path" : "/path/to/target",
  // 搜索设置
  "search": {
    // 基本没用，除非下面的 keyword 段匹配不到字段，才会用到它
    "defaultKey": "nm",
    // 用户输入的关键字，到底对应什么字段名呢？
    //  =xxx : 精确等于
    //  ~xxx : 以关键字开始
    //  xxx  : 模糊搜索
    "keyword": {
      "=id": "^[\\d\\w]{26}$",
      "~nm": "^[a-z0-9]{10}$",
      "title": "^.+"
    },
    // 用户指定的主过滤键的键名
    // 当 filter 有 majorValue 时采用这个键过滤
    "majorKey": null,
    // 固定一定带的搜索条件
    "match": {}
  },
  // 用户指定的过滤条件
  "filter": {
    // 关键字，会根据 search 里面的设置，来决定采用哪个键来匹配
    "keyword": null,
    // 用户指定的复杂查询条件
    "match": null,
    // 主过滤值
    "majorValue": null
  },
  "sorter": {
    "nm": 1
  },
  "pageNumber": 1,
  "pageSize": 1000,
  //--------------------------------------------------
  // 下面的字段为动态运行时的数据
  //--------------------------------------------------
  // 数据
  "data": {
    "list" : [],
    "pager" : {...}
  },
  // 处理状态
  "status": {
    "reloading": false
  }
}
```
